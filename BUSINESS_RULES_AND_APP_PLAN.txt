3. USER ROLES
============================================================

Required roles:

1. Company Admin
2. Manager
3. Level 1 Agent
4. Level 2 Agent
5. Customer

Company Admin:

- Creates/manages managers
- Manages all users
- Manages products
- Views all orders
- Confirms payment received from managers
- Releases products
- Approves or records commissions
- Views complete network
- Views reports

Manager:

- Adds Level 1 Agents
- Can have maximum 6 Level 1 Agents
- Receives money from Level 1 Agents
- Gives collected money to Company
- Views own commissions
- Views Level 1 and Level 2 network below them

Level 1 Agent:

- Added under a Manager
- Adds Level 2 Agents
- Can have maximum 6 Level 2 Agents
- Receives money from Level 2 Agents
- Gives collected money to Manager
- Views own commissions
- Views Level 2 Agents below them

Level 2 Agent:

- Added under a Level 1 Agent
- Adds Customers
- Collects money from Customers
- Gives collected money to Level 1 Agent
- Views own commissions
- Views Customers added by them

Customer:

- Must login before buying/requesting product
- Can view product catalog
- Can request/order products
- Gives payment manually to Level 2 Agent
- Can view order status

============================================================
4. MLM STRUCTURE
============================================================

The MLM model is unilevel with fixed role levels.

Structure:

Company
  -> Managers
      -> Level 1 Agents
          -> Level 2 Agents
              -> Customers

Capacity rules:

- Company can have unlimited Managers
- Each Manager can add maximum 6 Level 1 Agents
- Each Level 1 Agent can add maximum 6 Level 2 Agents
- Each Level 2 Agent can add Customers

Important:

- The Company is not limited to 6 Managers.
- The 6-member limit starts from Manager downward.

============================================================
5. COMMISSION RULES
============================================================

Commission is fixed per new person added.

Commission is not calculated as a percentage of product sale.

Commission is not based on product price.

Commission should be generated only after Company/Admin confirms payment.
This avoids fake registrations creating commissions without real money received.

Final commission table:

New person added     Added by        Commission
------------------------------------------------------------
Level 1 Agent        Manager         Manager gets 1000

Level 2 Agent        Level 1 Agent   Level 1 gets 1000
                                     Manager gets 500

Customer             Level 2 Agent   Level 2 gets 1000
                                     Level 1 gets 0
                                     Manager gets 0

Detailed rules:

1. Manager adds Level 1 Agent

- Manager receives 1000 commission
- Commission is created only after company confirms payment

2. Level 1 Agent adds Level 2 Agent

- Level 1 Agent receives 1000 commission
- Manager above Level 1 receives 500 commission
- Commission is created only after company confirms payment

3. Level 2 Agent adds Customer

- Level 2 Agent receives 1000 commission
- Level 1 Agent receives nothing
- Manager receives nothing
- Commission is created only after company confirms payment

6. PAYMENT AND PRODUCT FLOW
============================================================

There will be no online payment page at first.

Payment will be handled manually by cash or UPI.

Flow:

1. Customer gives money to Level 2 Agent.
2. Level 2 Agent gives collected money to Level 1 Agent.
3. Level 1 Agent gives collected money to Manager.
4. Manager gives collected money to Company.
5. Company confirms money received.
6. Company releases the product.
7. Product is distributed back down the chain.

Product delivery path:

Company
  -> Manager
      -> Level 1 Agent
          -> Level 2 Agent
              -> Customer

The app should track these stages manually.

Possible order/payment statuses:

- ORDER_CREATED
- MONEY_COLLECTED_BY_LEVEL_2
- MONEY_TRANSFERRED_TO_LEVEL_1
- MONEY_TRANSFERRED_TO_MANAGER
- MONEY_RECEIVED_BY_COMPANY
- PRODUCT_RELEASED_BY_COMPANY
- PRODUCT_RECEIVED_BY_MANAGER
- PRODUCT_RECEIVED_BY_LEVEL_1
- PRODUCT_RECEIVED_BY_LEVEL_2
- DELIVERED_TO_CUSTOMER
- CANCELLED

11. PRODUCTION SAFETY RULES
============================================================

These rules protect the MLM model from breaking after the system goes live.

No hard delete:

- Users, orders, commissions, payments, and network records must not be hard deleted once they are involved in business activity.
- Use status changes instead of deletion.
- Historical data must remain available for audits and reports.

User statuses:

- ACTIVE
- INACTIVE
- SUSPENDED
- TERMINATED

Meaning:

- ACTIVE users can login and perform their role actions.
- INACTIVE users keep history but cannot add users or earn new commissions.
- SUSPENDED users are temporarily blocked.
- TERMINATED users are permanently removed from operations, but their history stays.

Role upgrades:

- A Customer can request to become Level 2 or Level 1.
- A Level 2 can request to become Level 1.
- A Level 1 can request to become Manager.
- Role changes require Company/Admin approval.
- Past orders and commissions must not be recalculated after upgrade.
- New commissions follow the new role only from the approval date.

Reassignment:

- Only Company/Admin can reassign users from one upline to another.
- Reassignment affects future commissions only.
- Past commissions must not automatically change.
- Every reassignment must create an audit log.

Inactive upline handling:

- If Level 2 leaves, their old customers remain historically linked.
- If Level 1 leaves, their Level 2 agents remain historically linked.
- If Manager leaves, their network remains historically linked.
- Admin can freeze or manually reassign the affected branch.
- Inactive users cannot create new downline users or earn future commissions.

Duplicate account protection:

- Phone number should be required and unique.
- Email can be optional.
- Admin should be able to identify/handle duplicate account disputes later.

Commission ledger:

- Commissions must be ledger entries.
- Do not silently edit old commission rows.
- Corrections should be made through adjustment entries.

Commission statuses:

- PENDING
- APPROVED
- PAID
- CANCELLED
- ADJUSTED

Payment handover ledger:

- Manual money movement must be tracked step by step.
- Each handover should record from user, to user, amount, status, date, and optional proof.

Payment handover statuses:

- PENDING
- HANDED_OVER
- RECEIVED
- DISPUTED
- CANCELLED

Audit logs:

- Important actions must be logged.
- Examples: user creation, role upgrade, reassignment, payment confirmation, commission generation, commission adjustment, product release.

Beta matrix safeguards:

- Each Manager can have only one Beta Manager.
- Beta matrix completion counts only confirmed customers.
- Cancelled or unpaid customers do not count.
- Main Manager's 500-per-customer matrix commission remains pending until all 216 customers are confirmed.
- Once the Beta Matrix reaches 216 confirmed customers, release 108000 to the main Manager.
- After completion, the matrix is locked.
- No second Beta Manager or second matrix can be created under the same Manager.

Final production rule:

- Network placement is permanent for history.
- Changes are allowed only through admin-approved upgrade or reassignment records.
- Past commissions never change automatically.
- Future commissions follow the new approved structure.
